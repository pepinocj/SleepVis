find

Execute a MONGO QUERY on the server and finds documents

FIND is only capable of finding documents in a single COLLECTION. It is not possible to FIND embedded documents across multiple COLLECTIONS.

documents = find(mongodbconn, collectname)

documents = find(mongodbconn, collectname, 'Name', 'Value')
  • mongodbconn: Mongo Database Object
  • collectname: Collection name (as a MATLAB CHAR)
  • Different PARAM-VALUE pairs can be:
   Query:        A JSON-style MONGO Query used to filter documents in a collection and return only those who meets the criteria in Query. E.g., {"artist": "David"}
   Projection:   A JSON-style MONGO Projection key used to specify which KEY(s) should be projected in documents which are returned. E.g., {"song":1.0}
   Sort:         A JSON-style MONGO Sort key used to specify which KEY(s) should be used to sort documents which are returned. E.g., {"_id": 1.0}
   Skip:         A double value which skips specified number of documents from start in the resultset before it is received in MATLAB.
                 E.g., if a Query returns 10000 documents, SKIP of 1000 means, only last 9000 documents will be retrieved
   Limit:        A double value which limits number of documents to be retrieved. E.g., if a Query returns 10000 documents, LIMIT of 1000 means, first 1000 documents will be retrieved
   BatchSize:    A double value which determines number of documents to be sent in a single batch from database to MATLAB.
                 E.g., if a Query returns 10000 documents, a batchSize of 100 will get 100 records at a time. Will make 10000/ 100 trips between database and MATLAB

Example

documents = find(mongodbconn, 'product')
% finds and returns all documents present in collection 'product'

documents = find(mongodbconn, 'product', 'Query', '{"artist":"David"}')
% finds and returns all documents present in COLLECTION 'product' which has VALUE 'David' for KEY 'artist'.

documents = find(mongodbconn, 'product', 'Projection', '{"song":1.0}')
% finds and returns all documents present in COLLECTION 'product'.
% Each document retrieved will only have one KEY-VALUE pair present, i.e. SONG and its corresponding VALUE.

documents = find(mongodbconn, 'product', 'Query', '{"artist":"David"}', 'Projection', '{"song":1.0}')
% finds and returns all documents present in COLLECTION 'product' which has VALUE 'David' for KEY 'artist'.
% Each document retrieved will only have one KEY-VALUE pair present, i.e. SONG and its corresponding VALUE.

documents = find(mongodbconn, 'product', 'Query', '{"artist":"David"}', 'Sort', '{"_id":1.0}')
% finds and returns all documents present in COLLECTION 'product' which has VALUE 'David' for KEY 'artist'.
% Documents are SORTED using '_id' KEY.

documents = find(mongodbconn, 'product', 'Query', '{"artist":"David"}', 'Limit', 10)
% finds and returns first 10 documents present in COLLECTION 'product' which has VALUE 'David' for KEY 'artist'.

documents = find(mongodbconn, 'product', 'Query', '{"artist":"David"}', 'Skip', 10)
% finds and returns n-10 documents present in COLLECTION 'product' which has VALUE 'David' for KEY 'artist'.
% SKIPS first 10 documents.

documents = find(mongodbconn, 'product', 'Query', '{"artist":"David"}', 'BatchSize', 10)
% finds and returns all documents present in COLLECTION 'product' which has VALUE 'David' for KEY 'artist'.
% But, gets 10 documents in one trip to avoid memory overflow.
% Thus, total number of trips made is (TOTAL_DOCUMENTS/ BatchSize).
